# I18n

## Configuracio

- `src/i18n/i18n.js` registra `ca`, `es`, `gl`, `eu`.
- `fallbackLng` i `lng` inicial son `es`.
- Tambe es registren traduccions de `@somenergia/somenergia-ui` sobre la mateixa instancia.

## Implicacions practiques

- Un mateix flux pot tenir rutes localitzades diferents per idioma.
- Alguns `initialValues` depenen de `i18n.language`, per exemple `language: ${i18n.language}_ES`.
- El hook `useSyncLanguage(language)` canvia la llengua segons `useParams()`.

## Gotcha important

`src/i18n/i18n.js` inicialitza l'instancia global en importar-se. El propi fitxer admet que aquesta inicialitzacio es repetible i milloraria amb una `initI18n()`. No introdueixis una segona estrategia d'inicialitzacio sense revisar l'impacte global.

## Si canvies textos

1. Localitza les claus a `src/i18n/locale-*.json`.
2. Revisa si el component usa `t()` o rep text via props.
3. Si el text forma part d'una ruta, verifica les variants idiomatiques a `src/App.jsx`.
